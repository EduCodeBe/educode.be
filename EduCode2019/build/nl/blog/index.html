Educode engage des formateurs

Écrit par Nicolas Pettiaux le 2019-07-24

Formateur en technologies numériques et rédaction collaborative de contenus pédagogiques

Objectif: Former des enseignants aux outils et technologies numériques utilisables en classe.

Cela va de l'usage de logiciels tels que butineur internet (firefox, chromium), outils de gestion du courrier électronique (thunderbird, gmail ...), tableur, traitement de texte, base de données et logiciels de présentations (LibreOffice, LaTeX ...), logiciels de prise de vue et montage vidéo (gimp, inkscape, scribus) et son (audacity), programmation avec ou sans ordinateur (scratch, python, java ...), robotique (thymio, légo ...), impression 3D, découpage numérique, outils de calcul numérique, simulation ou représentation graphique (matlab, octave, scipy, geogebra, wims, labomep ...), rédaction collective à des outils spécifiquement adaptés à certains contenus pédagogiques (wiki, pads), à des outils de création d'horaires, de gestion de classe ou d'école, de mise à disposition de contenus pédagogiques et d'organisation de cours numériques (moodle, gepi, pmb ...)

Et aussi : participer au développement de ces outils (pour les étudiants en informatique)

QUESTIONS

  1. Quelles études suivez-vous ?
  2. Où et comment avez-vous appris ces outils et techniques ?
  3. Avez-vous déjà une expérience pédagogique ?
  4. Si oui, lesquelles ?
  5. Quelles sont vos expériences utiles ou intéressantes dans ce cadre ?
  6. Quelles sont vos attentes et souhaits ?
  7. À quel(s) niveau(x) et public(s) enseignez-vous ?
  8. Quels genres de formation ou information ou conférences espérez-vous en lien avec le numérique et l'informatique scolaire ?
  9. Dans quelle école enseignez-vous ? à quels niveaux ? où est l'école ?
  10. Quelles sont vos compétences en matière d'usages d'outils numérique à l'école ?
  11. Êtes-vous disponible le vendredi 27/9 toute la journée pour la 2è conférence educode pour y encadrer des ateliers ?
  12. Avec quels programmes informatiques êtes vous familiers ?

Joignez votre votre CV et un lien vers le site où vous publiez vos travaux

Merci d'envoyer un email à mailto:nicolas@educode.be contenant des notes de cours que vous avez écrites, des documents et œuvres que vous avez composés et tout élément pertinent pour montrer que vous maitrisez effectivement bien les technologies dont vous parlez. Des références vers les sites web que vous avez fait, les dépots de code auxquels vous avez contribués ... sont les bienvenus.

Contribuer à educode

Écrit par Nicolas Pettiaux le 2018-06-04

De meest technofiel van jullie (jullie allemaal ;-), die bij alle gelegenheden willen leren, waarderen het algoritme, een kleine code en wat erbij hoort, kan een terminal typen

  • installeren op uw machine (windows of macos x of gnulinux)
    • de gratis software waarmee de site gemaakt is Lektor (het is een stuk eenvoudiger om op een machine gnulinux te installeren dan macox x of windows, zonder dat het lastig voor ze is)
    • gratis collaborative work-software git
  • door een login aan te maken op github.com
  • Ga naar github.com/EduCodeBe/educode.be en doorzoek het project (rechtsboven)
  • het typen van de terminal (git-bash op windows)
    • git clone https://github.com//educode.be
    • cd /educode.be/EduCode
    • lektor-server
    • firefox http://localhost:5000/
    • (je kunt dan
      • bewerk de site met een mooie grafische interface in de browser en een klein potlood aan de rechterkant waarmee u kunt bewerken, en
      • ga bijvoorbeeld op de pagina http://localhost:5000/admin/root:exposanten/bewerken
      • voeg een "exposant" -pagina toe door op de eerste regel op + te klikken en een pagina toe te voegen waarvan het model "Exposant" is
      • door de velden in markdown format eenvoudig in te vullen
      • door lokaal op uw machine op te slaan onderaan de pagina
      • Als u een vertaling van een pagina wilt helpen, dan is dit waar het gebeurt door naar het menu aan de linkerkant van de taal te gaan. De teksten zijn al in het Spaans vertaald, maar ik heb geen tijd gehad om ze op te nemen.
      • de lokale server stoppen door CONTROL-C
  • door alles weer op de centrale server te zetten door te typen
    • git add *
    • git commit -m "een beschrijving van uw wijzigingen en toevoegingen"
    • git push (u voert dan uw logins en wachtwoorden in)
    • door mij te waarschuwen dat u wilt dat wij uw wijzigingen per e-mail integreren op site@educode.be
    • en indien mogelijk door op github een pull-request te doen (wat de bovenstaande en volgende procedures voor ons vereenvoudigt)
      • u gaat naar uw pagina https://github.com//educode.be/pulls
      • u klikt met de rechtermuisknop op de knop NEW PULL REQUEST en controleert of uw project gekoppeld is aan EducodeBe/educode.be (de master)
      • vul een korte notitie in waarin wordt beschreven wat je hebt gedaan
      • en verzenden
      • Ik accepteer uw voorstel als er geen conflict en hop is
      • Ik zet online zo snel als ik kan (of een ander lid van het team doet het) zoals dit, echt, samen implementeren we de samenwerking voor de site en de rest.

De praktische toepassing van theorie en digitaal is ook voor ieder van ons !

Un autre article de test

Écrit par Manuel le 2018-05-25

When the same comment thread appears on multiple pages, this is normally due to multiple thread identifiers being assigned to the same thread whenever the thread was initially created.

Resolving identifier conflict Identifier conflict can be resolved by verifying that a unique URL is being set and then assigning a unique identifier. This will create a new thread on pages which were previously displaying the same thread. Note that thread identifiers and URLs are set using our embed Javascript Variables.

You can also check if identifiers are being assigned to the same thread using our API console.

Prevention To prevent this from happening, you'll need to ensure that both the identifier and thread are unique when creating new threads.
